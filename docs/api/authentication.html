    Authentication API

    Reference Version v1 Stable Updated 03 Jun 2026

    Authentication endpoints issue and validate short-lived credentials for workload and user-scoped operations.

    This API is optional and supports advanced integrations. Most ULLR experiences are built directly on-device.

    POST /v1/session

    Exchange a project token for a scoped session.

    Field Type Required Description
    project string Yes Project identifier.
    scopes string[] Yes Granted permission scopes.
    ttl_seconds integer No Session lifetime, max 3600. 
    curl -X POST https://api.ullr.dev/v1/session \
  -H "Authorization: Bearer <project_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "project": "weather-console",
    "scopes": ["events:write"],
    "ttl_seconds": 900
  }'

    POST /v1/token/rotate

    Rotate an active machine token while preserving policy assignments.

    POST /v1/token/introspect

    Validate token health, scope grants, and expiration details.

    Errors

    Code Meaning Action
    AUTH_401 Token missing or invalid Check Authorization header and key status.
    AUTH_403 Scope denied Request broader scope from project policy owner.
    AUTH_429 Rate limit reached Apply exponential backoff and inspect quota settings.
