    Platform Overview
    Platform

    Platform overview

    ullr is a multi-device interaction platform. The phone runs the experience, the TV displays it, and interaction lives on the phone.

    Core concept

    Think of ullr as one shared room experience split across two screens. The phone handles logic and input. The TV is the shared display output.

    Mental model: phone equals engine and controller, TV equals display.

    Core architecture

    The system is designed around local, low-latency communication between devices on the same network. Rendering and input are coordinated on phone while TV playback remains lightweight.

    High-level architecture

    • Phone app: runs rendering, game logic, and controller input handling.
    • TV viewer: receives streamed output and presents the shared experience.
    • Transport: local-network streaming for low-latency screen output.

    Multiplayer model

    ullr supports multiple player phones joining the same session. Each phone acts as both a private screen and an input surface, while the TV remains the shared view of the game world.

    • Player 1 phone: private state + controller inputs.
    • Player 2 phone: private state + controller inputs.
    • TV output: synchronized shared scene for all players.

    Constraints

    Constraint Why it matters
    Same Wi-Fi network required Enables local pairing and reliable low-latency streaming.
    Phone handles rendering and processing Keeps compute on the player device and simplifies TV viewer requirements.
    TV is display-only Keeps setup simple and focused on shared-room viewing.
